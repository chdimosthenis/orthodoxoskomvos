# Review bot-seeded drafts

The daily-saints GitHub Action writes new entries to `src/content/saints/`
with `draft: true`. They're hidden from listings/RSS/today widget but live
at `/saints/<slug>/` for direct review. Each carries:
`sourceUrl`, `license: CC-BY-SA`, a body that links back to the source
article, and an `_(auto-seeded — please review)_` notice.

## Step 1 — list the drafts

```bash
grep -l "draft: true" src/content/saints/*.md
```

Read each one. Classify into:

1. **Real saint, content acceptable** → publish (remove the `draft` line)
2. **Real saint, content needs improvement** → rewrite body in original
   Greek prose, then publish
3. **False positive** (place name, navigation cross-ref, disambiguation,
   etc. that the parser misidentified) → delete the file

## Step 2a — publish as-is

Edit the markdown: remove the `draft: true` line entirely. The schema
defaults `draft` to `false` when unset.

If the bot's English content is acceptable for an /en/ entry, leave
`language: en`. If a Greek version is wanted, see the `translate-entry`
skill or rewrite per Step 2b.

## Step 2b — improve and publish

- Replace the auto-seeded body with original Greek prose (80–120 words)
- Set `language: el`
- Update `life` to a one-sentence Greek summary
- Verify `category` is correct (auto-seeder defaults to broad guesses
  based on heading keywords — "martyr"/"hierarch"/"monastic"/etc.)
- Optionally add `tropar`/`kontak` if a traditional Byzantine version exists
- Drop `sourceUrl` + `license: CC-BY-SA` if you've completely rewritten
  (the content is no longer derivative); set `license: original`
- Remove `draft: true`

## Step 2c — delete

```bash
rm src/content/saints/<slug>.md
```

Common false positives the parser produces:

- Place names (e.g. `patara.md`, `douai.md`)
- Date references (`march-19.md`)
- Disambiguation pages
- Generic categorical labels

## Step 3 — verify and ship

```bash
npm run build       # confirm nothing broke
git add . && git commit -m "review: published N saints, removed M false positives"
git push
```

## Don'ts

- Don't blindly remove `draft: true` on noise entries — delete them.
- Don't keep `sourceUrl: https://en.wikipedia.org/...` if you've
  completely rewritten the body. The attribution box exists to credit
  copied prose; on original prose it's misleading.
- Don't lose the bot's `wikipediaTitle` field on entries you keep —
  it stays useful for future `fetch_icon.py` runs.
- Don't review more than ~10 drafts per commit; smaller batches make
  the diff legible.
